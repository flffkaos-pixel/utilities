#️⃣ 해시(Hash)란 무엇인가?

해시는 임의의 길이를 가진 데이터를 고정된 길이의 문자열로 변환하는 함수입니다. 원본 데이터가 조금만変わっても 해시값은 완전히 달라지며, 역으로는 원본 데이터를復元할 수 없습니다. 이러한 특성은 비밀번호 저장, 데이터 무결성 검증, 디지털 서명 등에 필수적으로 활용됩니다.

🔐 주요 해시 알고리즘

MD5: 128비트 해시값을 생성합니다. 빠른 속도가 장점이지만 보안적으로는 취약하여 비밀번호 저장에는 사용하지 않습니다.

SHA-1: 160비트 해시값. 과거 HTTPS 인증서에 사용되었으나 현재는 Deprecated되었습니다.

SHA-256: 256비트 해시값. 현재 가장 널리 사용되는 알고리즘 중 하나로, 블록체인, SSL/TLS 등에 적용됩니다.

SHA-512: 512비트 해시값. 더 긴 해시값이 필요할 때 사용됩니다.

💼 실무에서의 활용

백엔드 개발에서 사용자 비밀번호는 반드시 해시화하여 저장해야 합니다. 평문 비밀번호를 저장하면 유출 시 모든 계정이 위험해집니다. bcrypt, scrypt, Argon2 등의 느린 해시 알고리즘이 비밀번호 저장에 권장됩니다.

파일 다운로드 시 무결성 검증을 위해 SHA-256 해시값을 제공하기도 합니다. 사용자가 다운로드한 파일의 해시값과 게시된 해시값을 비교하면 변조 여부를 확인할 수 있습니다.

#️⃣ What is a Hash?

A hash function converts data of arbitrary length into a fixed-length string. Even slight changes in original data produce completely different hash values, and original data cannot be recovered from hashes. These properties are essential for password storage, data integrity verification, and digital signatures.

🔐 Major Hash Algorithms

MD5: Generates 128-bit hash values. Fast but cryptographically weak—no longer used for passwords.

SHA-1: 160-bit hash. Was used in HTTPS certificates but is now deprecated.

SHA-256: 256-bit hash. One of the most widely used algorithms, applied in blockchain and SSL/TLS.

SHA-512: 512-bit hash. Used when longer hash values are needed.

💼 Practical Applications

In backend development, user passwords must be hashed before storage. Plaintext passwords put all accounts at risk upon breach. Slow hash algorithms like bcrypt, scrypt, and Argon2 are recommended for password storage.

File downloads often provide SHA-256 hashes for integrity verification. Comparing downloaded file hashes with published values confirms whether files are tampered.
