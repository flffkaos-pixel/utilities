📝 마크다운이란 무엇인가?

마크다운(Markdown)은 2004년 John Gruber에 의해作成된 경량 마크업 언어입니다. 간단한 문법으로 문서를 구조화할 수 있어, 웹 콘텐츠 작성, 기술 문서, 노트 테이킹 등广泛的 분야에使用됩니다. 특히 GitHub, Stack Overflow, Reddit 등 플랫폼에서 널리 지원하여 개발자 커뮤니티에서 표준처럼 사용되고 있습니다.

마크다운의 가장 큰 장점은 학습 곡선이 낮으면서도 강력한 표현력을 가진다는 것입니다. 별표(*)나 해시(#) 등의 간단한 기호로 제목, 목록, 링크, 이미지 등을 표현할 수 있어, HTML을 몰라도 웹 스타일의 문서를 쉽게 작성할 수 있습니다.

📖 기본 마크다운 문법

제목: #을 사용하여 H1~H6까지 표현 (# ~ ######)

강조: **텍스트** 또는 __텍스트__로 굵게, *텍스트* 또는 _텍스트_로 기울임

목록: - 또는 *로 unordered, 숫자. 로 ordered list

링크: [텍스트](URL) 형식

이미지: ![대체 텍스트](이미지URL) 형식

코드: `인라인 코드` 또는 ```블록 코드```

📝 What is Markdown?

Markdown is a lightweight markup language created by John Gruber in 2004. With simple syntax for structuring documents, it's used broadly in web content writing, technical documentation, and note-taking. Supported on GitHub, Stack Overflow, Reddit, and others—it's the standard in developer communities.

Markdown's biggest advantage is low learning curve with strong expressiveness. Simple symbols like asterisks (*) or hashes (#) express titles, lists, links, and images, allowing easy web-style documents without knowing HTML.

📖 Basic Markdown Syntax

Headings: Use # for H1~H6 (######)

Emphasis: **text** or __text__ for bold, *text* or _text_ for italic

Lists: - or * for unordered, numbers. for ordered

Links: [text](URL)

Images: ![alt text](imageURL)

Code: `inline code` or ```code block```

💼 Practical Applications

Markdown is essential for developers. Used in GitHub README.md, project docs, commit messages, and issue management. Technical blogs (velog, tistory, etc.) support Markdown editors, improving content writing efficiency.

Note apps like Notion, Obsidian, and Typora also support Markdown—write consistently across platforms with one syntax.
